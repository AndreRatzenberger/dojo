# T1 — Interface family to alert builder

You are given observations from four surfaces in one desktop operations product.

- Daily overview: charcoal field, strict twelve-column grid, near-square corners, hairline dividers, monospaced numerals, terse noun labels, dense tables without zebra striping. Lime appears only on live values and successful state changes.
- Incident detail: the same grid and density. Errors appear as a narrow vermilion edge beside the affected scope, never as a filled red card. A 120 ms snap confirms control changes; there is no bounce or blur.
- Empty state: the same geometry and typography, with one plain sentence and one outlined action. Empty space separates systems, not individual rows.
- Conference landing screen: full-bleed violet gradient, rounded cards, oversized slogan, illustrated mascot, and elastic motion. Release notes call it a temporary campaign surface; none of those choices recur elsewhere.

The downstream artifact is a new alert-configuration panel in the operations product. Another agent will design it.

Extract the product family's taste into a reusable, evidence-traced handoff. Weight recurrence and the campaign outlier honestly. Do not design the panel itself.
